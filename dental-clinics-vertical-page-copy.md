## 1. Quick layout review

**What works**

- 11-section flow is correctly ordered for a clinic owner arriving cold from outreach: hook → problem → definition → scope → proof → comparison → process → persona → objections → conversion → links.
- First paragraph after H1 is set up as the AI Overviews extraction target (R2).
- HomeCalc proof is bridged through YMYL (financial → dental), which is the strongest transfer-credibility move available.
- Comparison framed against the three options a dental clinic owner actually considers today (audit / dental SEO agency / GMB) per R3 — not generic categories.
- Pricing visible without overlapping the audit page's `Offer` schema (price stated as a signal, schema lives on the audit page).

**What should change before build**

- **Swap Sections 10 and 11.** Final CTA must be the last block on the page, not Related Resources. Use the same closing pattern as `/ai-visibility/ai-visibility-audit/` for system consistency.
- **Section 2 visual treatment.** Brief proposes "2–3 insight cards OR a bold scenario statement." Recommend one tight scenario block followed by short prose, not cards. Cards make the shift feel abstract; scenario + prose makes a clinic owner picture their own patient calling someone else.
- **Section 4 platform list vs check list.** Brief has both — keep both, but lead with **what we check for your clinic** (dental-specific, the differentiator) and let the platform list sit underneath. The platform list is the same across every vertical page; the check list is where this page earns its 60% uniqueness vs the mortgage brokers page.
- **No second case study placeholders in this section** (brief calls for two). Recommend one placeholder ("Dental clinic — case study publishing [Month] 2026") for now. Two empty cards on a vertical page read as more scaffolding than proof; one reads as "real work in progress."

**Fill-ins required before launch**
1. Walkthrough call duration (assumed 60 minutes — matches audit page).
2. Turnaround SLA (assumed 7 business days — matches audit page).
3. Month label for the dental case study placeholder.
4. Confirm city to use in the Section 2 example query — I used Toronto. If outreach is targeting a specific market, swap accordingly.
